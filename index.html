Let's say that you have n medical students who need to be assigned to n medical schools.

Each student attends 1 school and each school gets 1 student.

Each student has a list of which schools they prefer, and each school has a list of which students it prefers.

Can we construct an algorithm to match students to schools such that there are no cases where both a student and a school would prefer each other over the matches they have been assigned?

In other words, can we construct a matching with no unstable pairings?


Can we come up with an algorithm of O(n) to do so?

The Explanation of the Accepted Algorithm for this Problem is found below:























































Gale-Shapley Algorithm for Stable Matching

In this scenario, we are matching students with schools. The Gale-Shapley algorithm matches by having one group request the other, and the other either accepts or declines.

Suppose we have the students request the schools. The student will request their most-preferred school. If the school has not already accepted another student, then it will accept.

If the school has already accepted another student, then there are two options.

1) If the school prefers this student over the current student it has accepted, then it will abandon the old student and accept the new one.

2) If the school does not prefer this student over the current student, it will reject the student.



If a student has been rejected or abandoned, then they will choose their next prefered school to repeat.

This algorithm ensures that every student and every school each has a match, and that the matches are stable, since every student and school are matched in a manner so that there would not be a situation where both a student and school prefer each other over the matches they have. 


        stableMatching():
          init all s in students to free
          init all c in schools to free

          while(s is free):
            c = most-preferred school by s which s has not requested
            if c is free:
              s and c become paired
            else if c is not free (matched with s'):
              if c prefers s to s':
                s' becomes free
                s and c become paired
              else:
                s' and c remain paired
